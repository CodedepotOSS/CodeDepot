はじめに

CodeDepot では、ワード単位で検索を行います。 複数のワードは空白文字で区切って入力します。

以下にワードの例を示します。

  • random
  • RandomString
  • random_string
  • 乱数

以下の記号文字は特殊記号として扱います。

特殊記号 : " \ * ? - + & | ! ^ ~ ( ) { } [ ]

特殊記号を含むワードを検索する場合は、特殊文字の前に "\" 文字を入力します。

  • \(x\+y\)
  • X\:Y

以下のワードは予約後となっています。

予約語 AND OR NOT

ワード検索

CodeDepot では、以下のようなワード検索が行えます。

指定したワードを含むファイルを検索します。
厳密なワードの検索 "Random" 検索したいワードをダブルクォート文字(")で囲みます。
大文字/小文字を区別したより厳密なワードを検索します。
ワイルドカード文字による検索 test*
*len

該当するワードを含むファイルを検索します。

ワードの検索では大文字と小文字を区別せずに検索します。

検索フィールド指定検索

CodeDepot では、コードの特定の検索フィールドを指定して絞り込み検索が行えます。

検索フィールドを指定するには、検索文字列の前にフィールド名とコロン(:)文字を入力します。

以下に検索フィールド指定の例を示します。

  • cls:String
  • cls:test*
  • code:(open AND close)
  • -pkg:java.io.*

検索フィールドには、以下のフィールド名が指定できます。

フィールド名 説明
any 全ての検索フィールドを対象にします。
cls 指定したクラス名のコードを検索対象にします。
fdef 指定した関数名のコードを検索対象にします。
code コメント、予約語を除くコードを検索対象にします。
comment コードのコメントを検索対象にします。
pkg 指定したパッケージ名を検索対象にします。
prj 指定したプロジェクトを検索対象にします。
file 指定したファイルのパス名を検索対象にします。
license 指定したライセンス名を検索対象にします。
outType 指定した関数の返却値の型名を検索対象にします。
inTypes 指定した関数の引数の型名を検索対象にします。
fcall 指定した関数名を使用しているコードを検索対象にします。

複数の検索フィールドを同時に指定できます。

  • fdef:verify pkg:*.model
  • cls:Utils -pkg:org.apache.*

code 検索フィールドには、プログラムの予約語は含まれません。予約語を含めて検索する場合は、any フィールドを利用します。

  • any:static
  • any:"public static"

組み合わせ検索

ワード検索と検索フィールドを組み合わせて検索できます。

以下に組み合わせ検索の例を示します。

  • credit validate pkg:org.apache.*
  • base64 encode -license:GPL*
  • "throws *Exception" -cls:*Test
  • prj:(solr lucene) cls:Parser
  • fdef:(check OR verify) cls:entity

検索の詳細

CodeDepot の検索では以下の処理を行います。

  • 単純検索ならびにオプション検索では、空白文字や記号文字による区切りだけでなく、CamelCase による区切りを利用してワードを抽出しています。
  • ワードの検索では大文字と小文字を区別せず検索しており、英文のワードでは語尾変化を含んだワードも検索します。ワードをダブルクオート文字で囲んだ場合は、大文字と小文字を区別し、語尾変化を含まない厳密な検索を行います。
  • ファイルに日本語などのマルチバイト文字が含まれている場合は、ファイルの文字コードを判定し、マルチバイト文字２文字の連続をワードとして抽出します。例えば"変更依頼"という文字列は、"変更"、"更依"、"依頼"の３つのワードを抽出しています。
    注意事項
    ファイル内容によって文字コードが判定できない場合があります。
  • 検索結果やファイル内容では、検索文字をハイライトしますが、ハイライトするのは検索条件に一致した文字ではなく、検索条件に一致したファイル内に含まれる検索文字に一致する箇所全てをハイライトします。
